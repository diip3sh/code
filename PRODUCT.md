# Product

## Register

product

## Users

Developers using DP Code as a local web and desktop workspace for coding agents such as Codex, Claude, Gemini, Kilo Code, and OpenCode. They are often inside an active coding session, reading streamed output, reviewing changes, switching models, and managing local workspace state.

## Product Purpose

DP Code gives coding agents a minimal, reliable GUI with predictable session control, transcript rendering, local workspace context, and desktop packaging. Success means users can keep agent work moving without losing state, fighting reconnects, or interpreting unclear runtime status.

## Brand Personality

Quiet, technical, direct. The interface should feel like a dependable developer tool, not a marketing surface or decorative dashboard.

## Anti-references

Avoid oversized SaaS dashboards, decorative cards, vague AI-assistant chrome, heavy gradients, ornamental motion, and inflated spacing that makes operational state harder to scan.

## Design Principles

- Prioritize dense clarity: show runtime state in compact, readable groups.
- Keep behavior predictable during failures, reconnects, restarts, and partial streams.
- Let product UI serve the workflow; avoid decoration that competes with code and transcript content.
- Reuse established component vocabulary so controls feel familiar across the app.
- Make important state legible at a glance without turning every value into a hero metric.

## Accessibility & Inclusion

Target accessible contrast, keyboard-operable controls, clear focus states, reduced-motion-safe transitions, and readable text at browser zoom levels used by developers in long sessions.
